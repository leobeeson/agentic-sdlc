# Framework documentation

Documentation about the agentic SDLC harness itself. None of this is copied into a target repository; it explains how the harness is designed and how to drive it.

- `methodology.md`: the operating model. The harness thesis, the 80 per cent problem, the seven axioms, the converge-diverge-converge pipeline, intent on two dimensions, and the two composition primitives.
- `getting-started.md`: how to install the harness into a repository, run setup, and drive the first run, either by stating an intent or by hand.
- `guides/recipes-and-magnitude.md`: how a recipe becomes an execution plan, including the four pruning criteria and worked compositions at three magnitudes.
- `guides/session-continuity.md`: how an initiative survives sessions, including the initiative catalogue, the focus note, the prime skill, and the `/clear` hygiene rule.
- `guides/data-engineering-profile.md`: the grounding agents, the two data-engineering generators, their artefacts, and the capped testing loop.
